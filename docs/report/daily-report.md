# 📍 데일리 리포트 등록

> 데일리 리포트 등록 매뉴얼입니다.
>
> 경로 : `Report > Daily Report`
>
> 요약 : 스토어 선택 → 필수 정보 기재 && 피드백 정보 기재 후 임시저장 또는 저장

---

## 1. 데일리 리포트 작성 (필수 정보)

데일리 리포트 등록 화면에서 아래 항목을 순서대로 입력합니다.

*예) 일자 `2026-07-10` · 법인 `HQ_ATS법인` · 스토어 `도산 스토어` 기준으로 작성*

![데일리 리포트 작성](/img/daily-report-01.png)

| 항목 | 설명 | 필수 여부 |
|---|---|:---:|
| **리포트 일자** | 리포트를 기록할 일자 선택 | **필수** |
|  |  |  |
| **작성자** | 로그인 계정 정보로 **자동 인식** (별도 입력 불필요) | 자동 |
|  |  |  |
| **법인** | 계정 권한에 부여된 법인만 드롭다운에 조회 | **필수** |
|  |  |  |
| **스토어** | 선택한 법인 하위의 권한 부여된 스토어만 드롭다운에 조회 | **필수** |
|  |  |  |
| **방문객 수** | 해당 일자 집계 방문객 수를 숫자로 입력 | **필수** |

:::tip 참고
- 1개 날짜에는, 1개의 리포트만 등록할 수 있습니다.
- 필수 정보들을 설정 / 기재해야만 '저장'이 가능합니다.
- <b>판매 전환율(%)</b>과 <b>객단가</b>는 일일 지표 정보 필드 중 <b>'총 판매 제품 수량, 총 판매 금액, 매출 건 수'</b> 등록 시 자동 환산되어 산출됩니다.
- 모든 일일 지표 정보는 <u>0 이상의 숫자만 입력 가능</u>하며, <u>음수와 소수점은 불가</u>합니다.
:::

## 2. 데일리 리포트 작성 (피드백 정보)

> <span style={{color: '#2e7d32', fontWeight: 'bold'}}>2-1. 특이사항 게시글</span>
>
> 특이사항이나, 리포트로 전달하고자 하는 내용에 대해 직접 입력하실 수 있습니다.
>
> 또한 게시글 내 이미지 첨부 및 하단 파일 업로드를 통해 첨부 파일 등록이 가능합니다.

>

## 3. 임시저장 또는 저장

> <span style={{color: '#2e7d32', fontWeight: 'bold'}}>3-1. 임시저장</span>
>
> <b><u>필수 정보와 게시글이 입력된 상태</u></b>에서, 리포트를 '임시'로 저장할 수 있습니다.
>
> 임시저장된 리포트는 이후 상세 내용을 업데이트하며 여러 번 반복 저장이 가능하고, 최종적으로 '저장'을 통해 확정합니다.

> <span style={{color: '#2e7d32', fontWeight: 'bold'}}>3-2. 저장</span>
>
> 모든 정보가 <b><u>정상적으로 기입된 상태에서 최종 '저장'</u></b>을 할 수 있습니다.
>
> 저장 이후, 관리자 '승인' 권한이 없는 계정에서는 추가 수정이 불가합니다.

---

## 4. 리포트 승인 ('승인' 권한을 가진 계정에 한함)

<img src="/img/daily-report-06.png" alt="리포트 승인" style={{maxWidth: '600px', width: '100%'}} />

> <span style={{color: '#2e7d32', fontWeight: 'bold'}}>4-1. 승인 처리</span>
>
> '저장'된 리포트에 대해 '승인' 권한을 가진 계정은 최종 승인 처리를 할 수 있습니다.

> <span style={{color: '#2e7d32', fontWeight: 'bold'}}>4-2. SAP 전송</span>
>
> 승인된 리포트 정보에 대해 SAP로 일부 정보(입점객 등)를 전송합니다.

:::tip 참고
SAP로 전송되는 일부 정보는 아래와 같습니다.
- 판매 로스 : 제품 코드 / 대체 상품 코드 / 수량
- 입점객
:::
